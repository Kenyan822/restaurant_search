# ホットペッパーグルメAPI 簡易リファレンス

本プロジェクト（レストラン検索アプリ）で利用する「ホットペッパーグルメAPI」の仕様をまとめたドキュメントです。
公式リファレンス: [https://webservice.recruit.co.jp/doc/hotpepper/reference.html](https://webservice.recruit.co.jp/doc/hotpepper/reference.html)

---

## 1. グルメサーチAPI (店舗検索)
レストランや居酒屋の情報を検索するためのメインエンドポイントです。

- **ベースURL**: `http://webservice.recruit.co.jp/hotpepper/gourmet/v1/`
- **HTTPメソッド**: `GET`

### 主要なリクエストパラメータ
GETリクエストのクエリパラメータとして指定します。

| パラメータ名 | 必須 | 説明 | 指定例 |
| :--- | :---: | :--- | :--- |
| `key` | ⭕️ | APIキー | `HOTPEPPER_API_KEY`環境変数 |
| `format` | - | レスポンス形式（`xml` / `json` / `jsonp`）※デフォルトはXML | `json` (本プロジェクトではJSONを使用) |
| `keyword` | - | フリーワード検索（店名、住所、駅名など）。複数指定する場合はスペース区切り | `居酒屋 安い 新宿` |
| `party_capacity` | - | 宴会収容人数（指定した人数以上収容可能なお店を絞り込む） | `20` (20人以上の場合) |
| `count` | - | 取得する最大件数（デフォルト10件、最大100件） | `5` |
| `lat` / `lng` | - | 緯度 / 経度（現在地周辺の検索に使用） | `35.6895`, `139.6917` |
| `range` | - | 検索半径（1: 300m, 2: 500m, 3: 1000m, 4: 2000m, 5: 3000m） | `3` |

※LLMからの抽出情報（人数・キーワードなど）をこれらパラメータにマッピングして検索します。

### 主要なレスポンスフィールド
本プロジェクトで特に使用するフィールドです。（`data.results.shop` 配列内の各オブジェクト）

| フィールド名 | 説明 | 備考 |
| :--- | :--- | :--- |
| `id` | 店舗ID | `J000000000` |
| `name` | 店舗名 | |
| `address` | 住所 | |
| `genre.name` | お店のジャンル名 | 例: `居酒屋` |
| `catch` | お店のキャッチコピー | 評価(星)の代わりに表示するアピールポイント |
| `budget.name` | ディナー予算 | 例: `2001～3000円` |
| `capacity` | 総席数 | |
| `urls.pc` | ホットペッパーの店舗詳細ページURL | 予約リンクとして使用可能 |
| `photo.pc.l` | 店舗画像（大サイズ）のURL | サムネイルとしてUIで表示 |

## 2. 【重要】エラーハンドリングの注意点

ホットペッパーAPIは、**パラメータエラーや認証エラーの際にも HTTPステータスコード `200 OK` を返します**。

そのため、`fetch` や `axios` のステータスチェック (`response.ok`) だけではエラーを検知できません。レスポンスの中身を確認してエラー判定を行う必要があります。

**エラー時のレスポンス例 (JSON形式)**:
```json
{
  "results": {
    "api_version": "1.30",
    "error": [
      {
        "code": 3000,
        "message": "keyは必須パラメーターです"
      }
    ]
  }
}
```

- **1000**: サーバ障害エラー
- **2000**: APIキーまたはIPアドレスの認証エラー
- **3000**: パラメータ不正エラー

**実装時の対応**: `data.results.error` が存在するかどうかを確認するロジックが必要です。
