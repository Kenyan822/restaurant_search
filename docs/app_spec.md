# レストラン検索アプリケーション 全体仕様書

## 1. アプリケーションの概要
本アプリケーションは、学生やサークルなどの団体向けに設計された**「自然言語で検索できるレストラン検索サービス」**です。
ユーザーが「新宿で20人で入れる個室のある居酒屋を探して」のような自然言語のプロンプトを入力するだけで、AI（LLM）が条件を正確に抽出し、条件に合致した最適なレストラン情報を提案します。

- **ターゲットユーザー**: 幹事、学生、サークル代表者、複数人で食事をする予定のある人
- **主要な価値**: 従来のポータルサイトのような細かな条件設定（チェックボックスやプルダウン）を省略し、チャット感覚で手軽にお店を見つけることができる。

---

## 2. システム構成・技術スタック

本アプリケーションは、フロントエンドとバックエンドが統合されたモノリス構成を採用し、コンテナ化してクラウドにデプロイされています。

### 🎨 フロントエンド
- **言語**: HTML5, CSS3, JavaScript (Vanilla JS)
- **特徴**: 
  - シンプルかつ直感的なUI（検索バーと結果のカード表示）
  - バックエンドAPIとの非同期通信（`fetch` API）
  - レスポンシブ対応

### ⚙️ バックエンド
- **言語・フレームワーク**: Node.js, Express.js
- **役割**: 
  - 静的ファイル（フロントエンド）の配信
  - LLMを用いた意図抽出ロジックの実行
  - 外部レストランAPIとの連携とデータ整形

### 🤖 外部API・サービス
- **LLM API**: **Google Gemini API** (`gemini-2.5-flash`)
  - 用途: ユーザーの自然言語プロンプトから構造化データ（場所、人数、ジャンルなど）を抽出。
- **検索 API**: **ホットペッパーグルメ Webサービス API**
  - 用途: LLMで抽出した条件をもとに、実際の店舗情報、画像、空き状況、詳細情報を取得。

### ☁️ インフラ・デプロイ
- **ホスティング**: **Google Cloud Run**
  - コンテナ（Docker）ベースのサーバーレスコンピューティング環境を使用し、スケーラブルかつセキュアにホスティング。

## 3. アプリケーションの動作フロー

```text
[ユーザー]
   │
   ├── (1) 自然言語で検索 ("新宿で20人で入れる...")
   ▼
[フロントエンド (ブラウザ)]
   │
   ├── (2) POST /api/search
   ▼
[バックエンド (Express)]
   │
   ├── (3) プロンプトをGeminiへ送信
   ▼
[Gemini API (LLM)]
   │
   ├── (4) 構造化データ (場所, 人数, ジャンル等) を抽出して返却
   ▼
[バックエンド (Express)]
   │
   ├── (5) 構造化データをもとに検索クエリ発行
   ▼
[ホットペッパーAPI]
   │
   ├── (6) 該当するレストラン情報(JSON)を返却
   ▼
[バックエンド (Express)]
   │
   ├── (7) データ整形 (必要な情報のみ抽出し、共通フォーマットに変換)
   ├── (8) フロントエンドへ整形済みデータ(JSON)を返却
   ▼
[フロントエンド (ブラウザ)]
   │
   ├── (9) レストラン情報をカード形式で画面に描画して表示
   ▼
[ユーザー]
```

## 4. API インターフェース

フロントエンドとバックエンド間でやり取りされる内部APIの仕様です。

### 検索エンドポイント
- **URL**: `POST /api/search`
- **概要**: プロンプトを受け取り、条件に合致するレストランのリストを返します。

#### リクエスト
```json
{

}
```

## 5. 今後の拡張・改善プラン (ロードマップ)

本アプリはハッカソン向けに短時間で構築された MVP (Minimum Viable Product) であり、今後以下の機能追加を想定しています。

1. **レビュー評価の正確化 (Google Places API 連携)**
   - 現在の評価（星の数）はモック値（3.5〜4.8）を使用しています。今後は取得した店舗名と住所をもとに Google Places API を叩き、本物のクチコミ点数を表示します。
2. **メニューの要約機能 (スクレイピング)**
   - 店舗詳細URLからメニュー情報をスクレイピングし、LLMに「代表的なメニュー」や「おすすめポイント」を要約させて表示します。
3. **地図（Google Maps）との連携**
   - 検索結果にマップビューを追加し、ユーザーがお店の場所を視覚的に把握できるようにします。
4. **現在地周辺からの検索**
   - ブラウザの Geolocation API を利用し、「ここから近いラーメン屋」といった検索を可能にします。
